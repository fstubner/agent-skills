# Charter: product-acceptance

## Failure mode countered

Builder self-grades “done” on a pretty demo that misses the brief.

## Owns

- Adversarial review vs PRODUCT.md / product-brief.md
- SHIP / CONDITIONAL / BLOCK for **product outcome**
- Requiring evidence from eng / design / ux / backend gates
- Blocking when app-tier brief is missing
- Enforcing builder ≠ acceptor

## Does not own

- Re-running full visual CSS audits (consumes design-critique)
- Choosing stacks (consumes stack-decision / eng-structure report)
- Writing the product brief (may scaffold template, then hand to user/PM)

## Evergreen checklist themes

- Wrong persona / missing primary user
- Primary job not completable
- Scope creep vs anti-goals
- Demo-ware (fake data hides empty/error)
- Self-grade without separate acceptance pass
- Multi-part system with architecture BLOCK or missing `ARCHITECTURE.md`
