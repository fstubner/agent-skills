# Charter: frontend-engineering

## Failure mode countered

LLM rawdogs vanilla HTML/CSS/JS monoliths, dual icon systems, and “matches Unknown/vanilla forever.”

## Owns

- Job → stack **decision procedure** (no React monoculture)
- Scaffold / structure matching job complexity
- Reuse-before-invent (one framework, one icon set, one styling paradigm)
- Deterministic structure smell scripts
- `stack-decision.md`

## Does not own

- Brand aesthetics, tokens, anti-slop visuals (frontend-design)
- Primary-job UX honesty (frontend-ux)
- Product brief (product-management)
- Final product SHIP (product-acceptance)

## Evergreen invariants (properties)

1. **Existing stack wins** — extend; never dual-framework.
2. **Greenfield: infer → propose one → confirm once** — never silent React; never silent vanilla for app-tier.
3. **Structure matches complexity** — multi-view apps are modular; demos may be single-file HTML.
4. **One icon system, one CSS paradigm** — second set is a defect.
5. **Justified stack** — written rationale tied to job + constraints.

## Dated recipes

Optional `recipes/` under the skill may list current scaffold CLIs. They expire; the decision procedure does not.
