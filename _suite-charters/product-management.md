# Charter: product-management

## Failure mode countered

LLM builds a plausible demo, invents users/scope, and optimizes for “looks like a product” without a contract.

## Owns

- Short interview when PRODUCT.md is missing or hollow
- Writing / updating `PRODUCT.md` (see SHARED_CONTRACT.md)
- MVP cut and anti-goals
- Register hypothesis (brand vs product) confirmed with user

## Does not own

- Framework or library choice (frontend-engineering)
- Visual tokens or aesthetics (frontend-design)
- Flow interaction polish (frontend-ux)
- Final SHIP (product-acceptance)
- Server schema (backend-engineering)
