# Three-layer architecture

```
Skills (judgment) → References (understanding) → Scripts (verification) → Self-correction
```

## Skills

Answer: what / when / why.  
Contain principles, decision procedures, routing, stop conditions, interview strategy.  
Not long tutorials or “run every script” checklists.

## References

Answer: what good looks like and why.  
Patterns, anti-patterns, rationale. Loaded on demand by skills.

## Scripts

Answer: did measurable properties hold?  
Missing PRODUCT.md, dual frameworks, absent ARCHITECTURE.md, structure smells, report verdicts.  

Scripts **verify**; they do not replace judgment about taste, interview quality, or “best” architecture.  
Agents should: run → read report → repair → re-run.
