# UI guardrails — CI validation for layout regressions

Generated by `init-ui-guardrails.js` from the frontend-design skill.

## Quick start

1. Start your app locally (static server or dev server).
2. Run checks:

```bash
node scripts/ui-check.js --base-url http://localhost:PORT
node scripts/ui-check.js --base-url http://localhost:PORT --strict   # CI mode
```

## Config

Edit `ui-guardrails/fragile-surfaces.json`:

- **shell** — `sidebar` or `topbar` (layout assertion preset)
- **selectors** — map to your DOM (`.sidebar-pane` → your class names)
- **surfaces** — routes to hit; set `"layout": true` for invariant checks
- **viewports** — desktop / tablet / mobile sizes
- **smoke** — interaction tests (visibility, clicks, attrs)

Add surfaces when you ship new fragile views (docs sidebars, search modal, changelog).

## CI

See `.github/workflows/ui-guardrails.yml`. Requires Playwright:

```bash
cd path/to/frontend-design/ab-harness && npm install
# or: npm install -D playwright && npx playwright install chromium
```

Set `FRONTEND_DESIGN_SKILL_ROOT` in CI if the skill is not at `~/.cursor/skills/frontend-design`.
